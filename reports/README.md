# Specmatic reports

These are **HTML reports generated by Specmatic** (`specmatic test`, report configured in
[`../specmatic.yaml`](../specmatic.yaml) v3 → `specmatic.governance.report.formats: [html, ctrf]`),
one per CI test job. Open any `.html` in a browser. **Both jobs are at 100% API coverage.**

`specmatic.yaml` sets `specmatic.settings.test.schemaResiliencyTests: all`, so each job's single
test run already covers **both** example-driven conformance **and** generative/boundary resiliency
— that's why there are two reports (one per app spec), not four.

The app exposes a Spring-Boot-Actuator-style **`/actuator/mappings`** endpoint (test-only, in `app.py`),
so Specmatic reads the app's real routes and reports **actual** coverage — every contracted operation is
confirmed to exist in the running app and is tested (100% Absolute Coverage), not just matched against the spec.

| Report | Spec | Coverage | What it covers |
|---|---|---|---|
| `contract_public.html` | contract_public.yaml | 100% | conformance (external examples: 3 roles + rejection + 400) + generative negative/boundary tests |
| `api_contract.html`    | api_contract.yaml    | 100% | every response of every endpoint — 200 / 400 / 401, LLM mocked — conformance + generative negatives |

**How 100% on the protected API is reached honestly:** the app's real API bearer-token auth
accepts the token configured in `API_BEARER_TOKEN` (declared once in `../specmatic.yaml` under
`securitySchemes`, which is how Specmatic sends it). External examples in `../examples_api/`
exercise the authenticated `200`/`400` paths with that token, and a *wrong* token (or none) to
exercise the real `401` path — so a single run covers both the authorized and the unauthorized
response of every secured endpoint, using the app's ordinary auth (no test-only bypass).

The report also lists **Missing in Spec** endpoints (the app has 52 `/api` routes; the contract
governs the core 6) — that scoping is documented and justified in
[`../CONTRACT_SCOPE.md`](../CONTRACT_SCOPE.md).

Regenerate locally: run the commands in [Setup → Run all the Specmatic tests](../readme.md#2--run-all-the-specmatic-tests),
then copy `build/reports/specmatic/test/html/index.html` to `contract_public.html` or
`api_contract.html` here after each job. In CI each job also uploads its
`build/reports/specmatic/test/html` as an artifact.
