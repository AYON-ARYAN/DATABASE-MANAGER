# Specmatic reports

These are **HTML reports generated by Specmatic** (`specmatic test`, config in
[`../specmatic.yaml`](../specmatic.yaml) → `report.formatters: [text, html]`), one per
CI test job. Open any `.html` in a browser.

| Report | Spec | Mode | What it covers |
|---|---|---|---|
| `contract_public-contract.html`   | contract_public.yaml | contract   | value-checked conformance (inline + 3 external examples) |
| `contract_public-resiliency.html` | contract_public.yaml | resiliency | generative negative/boundary tests (exercise the 400) |
| `api_contract-contract.html`      | api_contract.yaml    | contract   | authenticated + LLM-mocked endpoints (incl. /api/command) |
| `api_contract-resiliency.html`    | api_contract.yaml    | resiliency | generative negatives across the full API |

Regenerate locally: start the app + LLM stub, then
`bash scripts/gen_specmatic_reports.sh` (see `scripts/local_specmatic_boot.sh`).
In CI each job also uploads its `build/reports/specmatic/test/html` as an artifact.
